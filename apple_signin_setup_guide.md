# Supabase "Sign in with Apple" Native iOS Configuration Guide

When implementing native "Sign in with Apple" in SwiftUI using the `AuthenticationServices` framework, the Swift code doesn't require a specific Client ID. Instead, Apple handles authentication securely via the app's internal Bundle ID. 

However, **Supabase needs your Apple Developer credentials** to validate the secure tokens generated by your app. 

Follow these steps when you create your Apple Developer account to connect it with Supabase.

## 1. Get Credentials from your Apple Developer Account
Once your [Apple Developer Account](https://developer.apple.com/) is set up, gather the following three pieces of information:

1. **Bundle ID:** This is your app's unique identifier. For this app, it is:
   `chronoslabs.loan-calculator`
2. **Team ID:** This is a 10-character string found in your Apple Developer account under your "Membership Details."
3. **Private Key (.p8 file) & Key ID:** 
   * Navigate to **Certificates, Identifiers & Profiles** > **Keys**.
   * Click the `+` to create a new key.
   * Give it a name (e.g., "Supabase Auth Key") and enable the **Sign in with Apple** capability.
   * Click Continue, then Register.
   * Note down the 10-character **Key ID**.
   * **Download the `.p8` file** (Note: Apple only lets you download this once. Keep it safe).

*Note: Also ensure that your App ID (in the Identifiers section) has the "Sign in with Apple" capability enabled.*

## 2. Configure the Supabase Dashboard
Go to your Supabase Project dashboard.

1. Navigate to **Authentication** -> **Providers** -> **Apple**.
2. Toggle **Enable Sign in with Apple** to ON.
3. Fill in the required fields:
   * **Services IDs / Client ID:** For native iOS apps, enter your app's Bundle ID: `chronoslabs.loan-calculator`. *(Do not use a web Services ID here).*
   * **Team ID:** Enter your 10-character Apple Team ID.
   * **Key ID:** Enter the 10-character Key ID from the `.p8` key you generated.
   * **Secret Key:** Open the downloaded `.p8` file in a text editor and paste its **entire** contents here (including the `-----BEGIN PRIVATE KEY-----` and `-----END PRIVATE KEY-----` lines).
4. Click **Save**.

Once these steps are completed, your native iOS app will successfully be able to pass Apple ID tokens to Supabase for authentication!
